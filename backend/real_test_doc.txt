The Self-Reflective RAG Engine is a production-grade AI backend that implements the architectural principles of the Self-RAG paper using modern open APIs. Rather than fine-tuning a model from scratch, this project replicates the Self-RAG reasoning workflow through structured prompt engineering over Groq's LLaMA 3.3 70B model. The system uses a Supabase pgvector store as the retrieval backend and a Flask REST API as the serving layer. The four core reflection tokens are Retrieve, IsRel, IsSup, and IsUse. The Retrieve token decides whether external knowledge is needed for a given query. The IsRel token evaluates whether a retrieved passage is actually relevant. The IsSup token checks whether a generated answer is grounded in the retrieved passages. The IsUse token scores the final answer's usefulness on a scale of 1 to 5.
